**New**

- Diagnostic output respects `NO_COLOR` and a new `--no-color` flag. Forwarded daemon stderr (kotlinc / konanc) is stripped of ANSI when color is off, and non-TTY redirect output stays plain end-to-end. (#2)
- `kolt <typo>` and `kolt --typo` emit a `Did you mean ...?` hint with the closest match when one exists in the command or flag set. (#2)
- `kolt fetch` / `kolt update` / `kolt add` print resolver progress as coordinates download — `[N/M] group:artifact:version` per coordinate, plus `  -> retry against <repository>` when a 404 falls through to the next repo. (#404)

**Behavior**

- TOML parse errors include the line number and, for unknown keys, a `Did you mean ...?` hint against the section's known keys. Watch-mode preserves the enrichment. (#2)
- `kolt new` with no flags opens an interactive 4-preset menu (`jvm app` / `jvm lib` / `native app` / `native lib`). Selecting a native preset triggers a follow-up native-target prompt. (#412)
- `kolt new` prompts use one option per line, numeric-only input, plain `>` arrow, and color-coded options (cyan for JVM defaults, yellow for native). The target prompt groups native options under a `-- native --` separator. (#407)
- `kolt init` prints the same next-step block as `kolt new`, so first-run guidance is consistent across both entry points. (#407)
- `install.sh` shows download progress for the kolt binary fetch instead of running silent. (#406)

**Installation**

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```
